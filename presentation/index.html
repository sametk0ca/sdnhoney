Project Overview

üéØ What is this project?

An advanced Software-Defined Network (SDN) honeypot system that uses machine learning to detect and redirect malicious traffic in real-time. The system simulates a network environment with multiple servers and honeypots to study cyber attack patterns.

ü§î Why did I choose this?

Cybersecurity is a critical field requiring innovative approaches. Traditional security methods are reactive, but this project demonstrates proactive threat detection using cutting-edge SDN technology and AI/ML integration.

‚ö° What does it do?

The system monitors network traffic, analyzes requests using ML algorithms, classifies threats in real-time, and automatically redirects suspicious traffic to honeypots for detailed analysis and logging.

Technical Implementation

Python 3.10 Ryu SDN Controller Mininet Network Emulation Flask Web Services Machine Learning OpenFlow Protocol REST API Real-time Analytics

üèóÔ∏è SDN Controller

Ryu-based controller with OpenFlow 1.3 support, implementing intelligent flow table management and real-time traffic redirection with priority-based rules (200-0 priority levels).

üß† ML Integration

Binary classification model (1=malicious, 0=benign) analyzing request frequency, user agents, and behavioral patterns with configurable risk thresholds.

üï∏Ô∏è Network Topology

Tree topology with 7 switches and 6 hosts including normal servers, triage honeypot, deep honeypot, and external traffic source simulation.

üìä Monitoring System

Comprehensive logging with JSON-structured data, real-time API endpoints, and detailed traffic analysis with ML predictions and risk scoring.
